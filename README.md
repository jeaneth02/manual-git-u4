# Manual de Git - Unidad 4
## 📝 Descripción
Este repositorio documenta la implementación y práctica de los 7 comandos fundamentales de Git (`init`, `status`, `clone`, `add`, `commit`, `push`, `pull`) para la sincronización de proyectos locales con GitHub. Sirve como evidencia de aprendizaje para la Unidad 4.

## 🛠️ Comandos Documentados
| Comando | Función Principal |
| :--- | :--- |
| `git init` | Inicializa un repositorio local vacío |
| `git status` | Muestra el estado de los archivos (modificados/nuevos) |
| `git clone` | Descarga una copia completa de un repositorio remoto |
| `git add` | Prepara archivos para el commit (Staging Area) |
| `git commit` | Guarda los cambios en el historial local |
| `git push` | Sube los cambios al repositorio remoto (GitHub) |
| `git pull` | Actualiza el repositorio local con cambios remotos |

## 📂 Estructura del Proyecto
- `README.md`: Documentación del proyecto.
- `.git/`: Carpeta interna de control de versiones.

## 🚀 Uso Básico
1. Clonar el repositorio: `git clone <URL>`
2. Hacer cambios y preparar: `git add .`
3. Guardar cambios: `git commit -m "mensaje"`
4. Sincronizar: `git push origin main`
